Таблица Country: позволяет хранить уникальные страны, что упрощает их обновление и связь с другими таблицами.
Таблица Brand: бренды связаны с конкретными странами, что упрощает хранение информации о происхождении бренда.
Таблица Color: уникальные значения цветов, что снижает дублирование данных.
Таблица Model: хранит модели автомобилей, объем потребляемого топлива и связывается с брендами, исключая повторение моделей с одинаковыми названиями.
Таблица Customer: отдельная информация о покупателях, имя и фамилия, телефон.
Таблица Sale: включает связи с моделями, цветами и покупателями, что упрощает выполнение аналитических запросов.

Такой подход обеспечивает наивысшую нормализацию данных и позволяет минимизировать дублирование информации.

Дополнительные пояснения по выбранным типам данных:
VARCHAR для строковых данных: выбрано для хранения названий стран, брендов, цветов и имен покупателей, поскольку они имеют переменную длину. Ограничения на максимальную длину (например VARCHAR(50)) указаны с учетом возможных значений.
INT для внешних ключей: выбран для всех внешних ключей (country_id, brand_id, model_id, color_id, customer_id) для связи с основными таблицами. Это стандартный тип для представления связей в реляционной БД.
DATE для даты продажи: это специализированный тип для хранения дат.
NUMERIC(n1, n2) для объема потребления топлива и цены: выбран для точного хранения числовых значений с двумя знаками после запятой.
